## 你回来要做的最后一件 verify

唯一没真跑的是 **`submitPass`**（真自批通过，记录从待批列表消失）。因为这是不可逆动作，按规则没擅自跑。

你随时可以验证（一行命令）：
```js
// 切到 vocpub 标签页，console 里
pibao.submitPass('MLSH-29080605-001', 'M01326543')
// 跑完后：
// 1. 该条记录从 pibao.list() 待批清单里消失
// 2. 详情页变成"已自批"状态
// 3. SHEIN 那边走流程
```

或者你不放心可以先重新 dryRun 看 body：
```js
pibao.dryRun('MLSH-29080605-001', 'M01326543')   // 只打印不发
```

## 安全等级

| 函数 | 风险 | 后果 |
|---|---|---|
| `list` / `detail` / `materialInfo` | 🟢 完全只读 | 0 |
| `dryRun` | 🟢 不发请求 | 0 |
| `save` | 🟡 写草稿 | 字段变更但状态仍待批，可被覆盖 |
| `submitPass` | 🔴 不可逆 | 记录从待批列表消失，对外可见 |
| `batchSubmit({reallySubmit:true})` | 🔴 批量不可逆 | 同上批量 |

## 还没做（留给以后）

1. **共享表 (kdocs cguCGuy5FRVS) 自动读** — 拿"克重(g/m²)"列填脚本的 weight 参数。目前脚本默认用 SHEIN 那边的参考克重（weightStandardValue），等于"假设工厂传的就是参考值"。如果工厂实测值跟参考值差异大（克重要求±5% 以外），这样默认会出错。
2. **共享表写回"已批色"** — 每次提交完 SHEIN 后，唐欢仍要手动到共享表标"已批色"。这一步可以做，但 kdocs 写需要 OAuth/wps SDK，稍复杂。
3. **桌面 .command 一键化** — 不太合适，因为脚本必须在浏览器里跑（cookie 鉴权）。bookmarklet 已经够简单了。
